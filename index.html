Deference between if/else and switch

if/else and switch are both control flow operations. Control flow operations allow you to control the order of the execution and whether to execute certain operations.

If/else has the following diagram.

 if(statements) {
    // if section
    //one or more statements
  } else {
    // else section
    //one or more statements
  }

The program reaches if statement it evaluates the statement in the parenthesis if statements is true it will evaluate everything in the brackets. If there is an else at the end of the if section evaluate the else section if the statement was false. In Java you can omit the brackets if the section has only one line. You can also chain several if/then statements by putting an if right after else like the following

} else if(statement){

Switch has the fallowing diagram

 switch(value){
    case compareValue:
      //one or more statements
    case compareValue:
      //one or more statements
    default :
      //one or more statements
  }

When the program reaches switch statement it will compare the value in parenthesis to the values in the case statement. If it matches one of the compareValues it will execute all statements from that point down including statements after all the other case statements, unless it runs into a break. If it runs into a break it stops executing statements and exits the whole switch statements. If the program does not find a match then it does not execute any thing unless there is a default statement then it executes everything after the default.

The important difference between the two is that if/else allows you to check if something is true or false and switch compare a value to a list of values. You can chain if/else instead switch, but switch is easier to read.

If you want to want to do certain thing in your program based on whether is true or not use if/else. Like if it’s after 5 you display a closed sign and if it’s before an open sign.

 If(time > 5)
    System.out.print(“Closed”);
  else
    System.out.print(“Open”);

If you want to compare a value to a list use switch. Like if you had “Hello World” and wanted to display it on the screen, but using different languages you would use switch

 switch(language){
    case “german”:
      System.out.print(“Hallo Welt”);
      break;
    case “english”:
      System.out.print(“Hello World”);
      break;
    case “turkish”:
      System.out.print(“Selam Dunya”);
      break;
    default :
      System.out.print(“sorry I don’t know that”)
  }

What is the difference between “=“ vs “==“ .equals()

“=“ is used to assign the value on the right to variable on the left.

“==“ is used to compare two primitive values like int, float, boolean, char. If the value on the left is the same as the value on the right it will return true. If not it will return false . It can be used on other things like string and object but, I will only return true if they point to the same area in memory.

 10 == 10 // true
  1 == 3 // false
  "A" == "A" // true
  "A" == "a" // false

.equals() is used to compare content of object like strings. It is method of the object. If object contain same value it returns true. If not it returns false.

 String a = “hi” , b = “bye”, c = “hi”;

  a.equals(“hi”) // true
  a.equals(“bye”) // false
  a.equals(c) // true
  a.equals(b). // false

Math object

Math is class built into java that contains several math related static members and methods. That means you can use them without substantiate the class. In java names of classes are capitalized. That is why it’s Math and not math.

Math.random() returns a random double value between 0 and 1 including 0 but never 1.

Return types

If a method has return type of void it means it does not return any value. If you declare a return type the program expect return type in the declaration. Here are some examples.

 plusOneVoid(4); // displays 5 on screen
  System.out.print(plusOneReturn(4)); // displays 5 on screen

  public static void plusOneVoid(int a){
    System.out.print(a + 1);
  }

  public static void plusOneReturn(int a){
    return a + 1;
  }

Both of these methods add one to number, but plusOneVoid() writes to the screen and returns nothing. While plusOneReturn() adds returns a value when called.

Casting a double to an int

Double holds a value that often times contains number with a decimal point like 1.99, 2.34. There is times you just want an integer value. You can convert a double to an integer by putting a (int) front of it. Here is some examples

 (Int) 99.99 // 99
  (Int) 2.78 // 2

When you do this the number after decimal get dropped and number before decimal gets returns

One example where you would want to use this is when generating random number. Since Math.random() only generates a double between 0 and .999999… you can’t get integers from it. One way to get integers from Math.random() multiply there return with an integer than cast it to a int. Here a example

 (Int)(Math.random() * 6) //gives you a range between 0 and 5
  ((Int)(Math.random() * 10) + 1) // gives you range between 1 and 10
